Barcode Types Guide

From Code 128 to PDF417: Understanding Barcode Types

Published: November 27, 2025

Not all barcodes are created equal. Choosing the right symbology can mean the difference between smooth operations and constant scanning failures.

Code 128 – The Universal Standard

The most widely supported and efficient 1D barcode. Perfect for:

  • Warehouse labeling
  • Inventory tracking
  • Shipping labels
  • Internal asset management

Supports full ASCII, numbers, and special characters. Highest density of any 1D format.

Code 39 – Legacy but Reliable

Older but still used in government, defense, and automotive. Only supports uppercase letters, numbers, and a few symbols — but extremely forgiving to print quality.

EAN-13 & EAN-8 – Retail Essentials

Global retail standard. EAN-13 is used on 99% of consumer products worldwide. AxelBase validates checksums automatically — no more “invalid barcode” rejections at checkout.

UPC-A – North American Retail

12-digit version of EAN used primarily in the US and Canada. Fully supported with checksum validation.

PDF417 – When You Need More Data

The only 2D barcode in AxelBase. Can hold up to 2000 characters — perfect for:

  • Driver’s licenses
  • Boarding passes
  • Certificates
  • Detailed product info

Quick Selection Guide

  • Retail products → EAN-13 / UPC-A
  • Warehouse & logistics → Code 128
  • Government / legacy → Code 39
  • Large data → PDF417

FAQ

Which is most scannable?

Code 128 and EAN-13 are equally reliable when properly sized.

Can I mix types in one batch?

Yes! AxelBase supports mixed symbologies in a single ZIP export.

The right barcode isn’t the newest — it’s the one your scanner understands.
